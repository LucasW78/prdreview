# 更新说明

## v0.1.3

### 本次更新
- 新增权限体系：超级管理员与业务线角色分权，业务线角色仅可访问所属模块的知识库上传/查询能力。
- 新增权限配置能力：支持在线维护 `super_admin_emails` 与 `business_line_members`，并持久化到数据库。
- 评审任务列表接口升级为分页返回，默认摘要模式（可选是否包含快照明细）。
- 知识库向量入库新增 `doc_type` 标记，检索支持按 `doc_type` 过滤；评审链路改为“优先 SOP，补充 PRD”。
- 评审检索升级为多探针策略（标题/规则句/摘要/全文融合召回），冲突命中率提升。
- 增加冲突规则兜底：命中“强制覆盖/必须”与历史“非强制建议”证据时自动补出 `logic_conflict`。
- 增强问答证据可视化：来源编号（S1/S2）、文档类型、章节信息联动展示。
- 增加评审解析失败日志能力：后端落盘 `analysis_parse_failures.jsonl`，并提供查询接口。
- 修复 Merge 跨模块误去重：去重条件由 `content_hash` 收敛为 `content_hash + module + doc_type`。
- 修复上传编码鲁棒性：`utf-8`/`utf-8-sig` 回退，编码不支持时返回 422 明确提示。

### 零风险精简
- 删除未被调用的同步问答冗余实现，仅保留异步问答主路径。
- 清理后端接口中的调试噪音输出（print/traceback），不改变业务行为。

## v0.1.2

### 本次更新
- 智能问答交互重构为 Gemini 风格：左侧会话历史、右侧会话区。
- 新增会话管理能力：新建、重命名、删除、清空全部、刷新后本地持久化。
- 新增 AI 回复“重新生成”能力，支持基于同一问题重试。
- 修复“无知识依据时仍显示参考来源”的问题，前后端双重兜底隐藏来源。
- 评审工作台 Merge 结果统一为页面弹窗提示，成功时支持“查看”并跳转对应知识库。
- 新增“提示词管理”独立模块（侧边栏入口），支持回显/编辑/应用评审系统提示词。
- 清理与项目业务无关代码：删除临时脚本、未引用组件及无用依赖。

### 清理项
- 删除文件：`patch_review.py`、`src/components/EditableTextBlock.tsx`、`.trae/`
- 移除前端无用依赖：`@google/genai`、`express`、`dotenv`、`motion`、`@types/express`

## v0.1.1

### 本次更新
- 修复 Merge 弹窗与实际提交内容不一致的问题，统一使用优化后的最终文档内容。
- 优化 Merge 失败提示，前端可展示后端返回的具体错误原因。
- 增加 Merge 前校验：缺少任务 ID 或提交内容为空时给出明确提示。
- 增强后端 Merge 异常处理与事务回滚，降低异常场景下的数据不一致风险。
- 增加知识库索引容错：索引失败不阻断 Merge 主流程，并返回索引失败信息。
- 调整归档文件命名规则：保存为 `${需求标题}.md`，并自动清洗非法文件名字符。
